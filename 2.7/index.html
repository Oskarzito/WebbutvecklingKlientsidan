2.7 Sidlayout

Webbsideslayout (fixerad, flytande och elastisk) (fixed, fluid, and elastic)

Fixed
- Storlekar har bestämda mått, vanligtvis angivna i pixlar ( px). Dessa hemsidor ser oftast bara bra ut på en skärmstorlek.
Fluid
- Storlekar anges i procent. När skärmstorleken förändras justeras hemsidans innehåll skalenligt. Detta kan dock se rätt dåligt ut exempelvis på mindre skärmar. 10% på en stor skärm kan vara rimligt. På en liten telefon blir detta innehåll väldigt litet.
Elastic
- Storlekar anges i em. Bra eftersom mått på hemsidans element skalas efter fontstorleken. Men om man ändrar fontstorleken kan hemsidans utseende ändras drastiskt.

Varför CSS istället för tabeller

Man bör använda CSS istället för tabeller när man skapar sina layouter. Detta eftersom det främst är semantiskt rörigt om man använder tabeller för att styra sin layout istället för att presentera data. Vidare är det mer flexibelt att använda CSS när man kan använda flexbox med mera. Tabeller bör användas till att presentera data, inte för layout. CSS är till för stilsättning, därav är det att föredra.
