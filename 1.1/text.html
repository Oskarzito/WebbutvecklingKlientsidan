Text och hypertext i HTML

Här går vi igenom hur text representeras i olika format på skärmen.

<blockquote>

Denna tag används när man vill citera något från exempelvis en annan hemsida. Som taggens namn indikerar används denna tag när man vill citera text-stycken. Exempelvis:

Java is a general-purpose computer-programming language that is concurrent, class-based, object-oriented, and specifically designed to have as few implementation dependencies as possible. It is intended to let application developers "write once, run anywhere" (WORA), meaning that compiled Java code can run on all platforms that support Java without the need for recompilation.
Ett blockquote skrivs som följande: 

            
        <blockquote cite="länk-till-källan-man-citerar-från">
            Textstycke som skall citeras skrivs här!
        </blockquote>

h-rubriker

Viktigt när man skriver text är att man skiljer på rubriker och vanlig text. En tag som kan användas för att representera rubriker är h-rubriker. h-rubriker skrivs som en helt vanlig tag i HTML; ett h följt av en siffra från 1 till 6, där 1 är största rubriken och 6 är minsta. Exempel på storlekar är följande:

<h1>h1-rubrik</h1>

<h2>h2-rubrik</h2>

<h3>h3-rubrik</h3>

<h4>h4-rubrik</h4>

<h5>h5-rubrik</h5>
<h6>h6-rubrik</h6>

Skulle man vilja, kan man med hjälp utav CSS ändra fonten, storleken och färgen hos rubriker.

Paragrafer

Paragrafer i HTML representerar text man vill ha ut på skärmen. Dessa skrivs med ett enkelt p, som följande:

<p>text man vill skriva ut</p>.

Skulle man vilja, kan man med hjälp utav CSS ändra fonten, storleken och färgen hos paragrafer.

<pre>

Taggen pre står för pre-formatted. Vad denna tag gör är att den presenterar text i det format som det står i källkoden direkt på skärmen. Vanliga <p>-taggar påverkas inte om man i källkoden skriver massvis av tab-tecken innan. Det gör däremot text som står inom <pre>-taggar. Nedan följer två exmepel, där det i källkoden är lika många mellanrum som separerar orden:

1.Text skriven som paragraf. 

            2.Text          skriven
                  som     pre-formatted.

<pre>-taggar är väldigt behändiga om man bland annat vill skriva ut källkod korrekt indenterad på skärmen.

<span>

span är en tag som lägger sitt innehåll direkt efter föregående element. Det är en så kallad inline-container. Denna tag är behändig när man vill style:a ett ord i en längre mening, eller om man har anledning att gruppera ihop något. Varför man använder span är eftersom element i HTML vanligtvis läggs under varandra, i samma ordning man skapar dessa i sin kod. 

            <p> Början på en mening </p> <p class="textnamn"> följt av slutet på en mening. </p>

Resulatet av denna HTML-kod är:

Början på en mening

följt av slutet på en mening.

Gör man detta med span istället, kan man inline:a sin nya text: 

            <p> Början på en mening </p> <span class="textnamn"> följt av slutet på en mening. </span>

Början på en mening följt av slutet på en mening.

På detta sätt kan man komma åt och modifiera andra halvan av sin mening via klassnamnet textnamn. Detta är inte endast begränsat till textparagrafer. Man kan applicera span på massvis av HTML-taggar för att få ett inline-beteende.
