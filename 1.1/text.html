Text och hypertext i HTML

Här går vi igenom hur text representeras i olika format på skärmen.

Text

<blockquote>

Denna tag används när man vill citera något från exempelvis en annan hemsida. Som taggens namn indikerar används denna tag när man vill citera textstycken. Exempelvis:

Java is a general-purpose computer-programming language that is concurrent, class-based, object-oriented, and specifically designed to have as few implementation dependencies as possible. It is intended to let application developers "write once, run anywhere" (WORA), meaning that compiled Java code can run on all platforms that support Java without the need for recompilation.
Ett blockquote-element skrivs som följande: 

            
        <blockquote cite="https://en.wikipedia.org/wiki/Java_(programming_language)">
            Textstycke som skall citeras skrivs här!
        </blockquote>

Attributet cite representerar den länk man hämtade sitt citat ifrån. Text skriven inom blockquote-taggar blir automatiskt indenterad av webbläsaren.

h-rubriker

Viktigt när man skriver text är att man skiljer på rubriker och vanlig text. En tag som kan användas för att representera rubriker är h-rubriker. h-rubriker skrivs som en helt vanlig tag i HTML; ett h följt av en siffra från 1 till 6, där 1 motsvarar största rubriken och 6 den minsta. Exempel på storlekar är följande:

<h1>h1-rubrik</h1>

<h2>h2-rubrik</h2>

<h3>h3-rubrik</h3>

<h4>h4-rubrik</h4>

<h5>h5-rubrik</h5>
<h6>h6-rubrik</h6>

Skulle man vilja, kan man med hjälp av CSS ändra fonten, storleken och färgen hos rubriker. För oss som läser text på skärmen är det egentligen ingen skillnad om man skriver en h1-rubrik och förstorar texten, eller skriver en h6-rubrik. Det handlar främst om semantiken.

Paragrafer

Paragrafer i HTML representerar text man vill ha ut på skärmen. Dessa skrivs med ett enkelt p, som följande:

<p>text man vill skriva ut</p>.

Skulle man vilja, kan man med hjälp av CSS ändra fonten, storleken och färgen hos paragrafer.

<pre>

Taggen pre står för pre-formatted. Vad denna tag gör är att den presenterar text i det format som det står i källkoden direkt på skärmen. Vanliga <p>-taggar påverkas inte om man i källkoden exempelvis skriver massvis av tab-tecken innan, och mellan, orden. Det gör däremot text som står inom <pre>-taggar. Nedan följer två exmepel, där det i källkoden är lika många mellanrum som separerar orden i både paragrafen samt pre-formatted-elementet:

1.Text skriven som paragraf. 

            2.Text          skriven
                  som     pre-formatted.

<pre>-taggar är väldigt behändiga om man bland annat vill skriva ut källkod korrekt indenterad på skärmen. Det är helt enkelt programmerarens formatering som används, inte webbläsarens.

<span>

span är en tag som inline:ar sitt innehåll med föregående element. Det är en så kallad inline-container. Denna tag är behändig om man exempelvis vill style:a ett enda ord i en längre mening, eller om man har anledning att gruppera ihop något inom ett HTML-element. Varför man använder span är eftersom element i HTML vanligtvis läggs under varandra, i samma ordning man skapar dessa i sin kod. Låt oss exemplifiera med en vanlig HTML-paragraf: 

            <p> Början på en mening </p> <p class="textnamn"> följt av slutet på en mening. </p>

Resulatet av denna HTML-kod är:

Början på en mening

följt av slutet på en mening.

Gör man detta med span istället, kan man inline:a sin nya text: 

            <p> Början på en mening <span class="textnamn"> följt av slutet på en mening. </span></p>

Början på en mening följt av slutet på en mening.

På detta sätt kan man komma åt och modifiera andra halvan av sin mening via klassnamnet textnamn. Detta är inte endast begränsat till textparagrafer. Man kan applicera span på massvis av HTML-taggar för att få ett inline-beteende. Observera att span inte gör text till kursiv. Detta gjordes endast för att förtydliga.

Hypertext

a

För webbsidor är det både nödvändigt och viktigt att man inte presenterar allt innehåll på en gång för klienten som surfar in på hemsidan. För att göra hemsidor mer användarvänliga använder man sig utav länkar.

Länkar i HTML gör man via taggen <a>. 

            <a target="_blank" href="här-specificerar-man-vart-man-vill-länka">
                Element som blir till en klickbar länk</a>

Taggen <a> ovan har två attribut, href och target. href pekar på den destination man vill att klienten ska hamna när denna klickar på länken. target i sin tur säger var länken ska öppnas. Detta attribut kan ha flera värden, _blank säger att när man klickar på länken ska den nya sidan (destinationen) öppnas i en ny flik.

Absoluta och relativa sökvägar

Vill man till exemepel länka till en HTML-sida kan man göra det på två sätt: genom att definiera en absolut sökväg till destinationen, eller en relativ sökväg. Om vi vill länka till startsidan på denna hemsida kan vi definiera detta enklast med en relativ sökväg enligt följande: 

            <a target="_blank" href="index.html">Klicka här för startsidan.</a>

Vilket blir till länken: Klicka här för startsidan.
index.html är en relativ sökväg. Relativa sökvägar börjar söka från den position där länken läggs in, för att sedan därifrån definiera vägen till destinationen. I detta fall ligger denna HTML-sida i samma projektmapp som index.html, därför behöver man endast skriva namnet på filen som href-destination. Hade denna sida legat i en undermapp än vad destinationen gjorde så hade vi varit tvunga att specificera vägen med ../index.html, som säger "hoppa upp ett steg i projektstrukturen, välj sedan index.html".

Till skilland från en relativ sökväg finns absoluta sökvägar. Absoluta sökvägar innehåller hela sökvägen från en hemsidedomäns rot (inklusive domännamnet) till destinationen. 

            <a target="_blank" href="https://stackoverflow.com/questions/tagged/html">
                Klicka här för StackOverflow.</a>

Vilket ger länken: Klicka här för StackOverflow.
Som vi ser pekar denna sökväg på domänen StackOverflow. Sedan pekar den på underkatalogen questions, följt av underkatalogen tagged, följt av underkatalogen html. Detta är en länk med en absolut sökväg. Dessa sökvägar är alltid lika, oavsätt varifrån man länkar, till skillnad från de relativa sökvägarna.

Länk till ankare

Ibland vill man inte bara länka till andra hemsidor, ibland vill man även kanske tillåta klienter att hoppa till olika delar på samma sida genom att exempelvis klicka på en rubrik eller dylikt. Detta kan göras med länkar till ankare.

Säg att vi råkat hamna här nere på hemsidan av misstag, och har ingen aning om vad länkar är för något. Detta kan vi lätt hjälpa till att reda ut genom att skriva: 

            <a href="#hypertext">Klicka här för att läsa om länkar.</a>

Detta ger oss länken: Klicka här för att läsa om länkar.
Genom att i href peka mot ett HTML-elements id kan vi instruera webbläsaren att hoppa till platsen det elementet befinner sig på en hemsida. Destinationen denna länk pekar på är rubriken Hypertext. Den är definierad enligt följande: 

            <h2 id="hypertext">Hypertext</h2>

Genom att definiera ett id för HTML-element kan vi i våra href-definitioner skriva ett #-tecken följt av destinationelementets id. Detta ger oss en länk till ett ankare.

Bild-länkar

Länkar är inte något som är exklusivt för text. Andra HTML-element kan även göras klickbara. Bilder kan vara av intresse att ibland kunna klicka på. En bild-länk skapas enkelt genom: 

            
                <a href="index.html#structure-header">
                    <img src="img/taggarHTML.jpg" alt="Taggar i HTML">
                </a>

Vilket ger:

Det man gör att att skapa ett länk-element med a-taggen. I detta element lägger man sedan in bilden. På detta sätt syns bilden, precis som den ska, men den även klickbar och går till den destination man specificerar i länkens href.
