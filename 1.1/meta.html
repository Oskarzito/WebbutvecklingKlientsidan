Metainformation i HTML

Meta-taggar i HTML definieras i ett dokuments huvud och innehåller information om data (så kallad metadata). Vad detta betyder är att taggarna ej visar innehåll på skärmen för klienten, utan snarare exempelvis hjälper en sökmotor att hitta till ens hemisda och dylikt. Här beskrivs meta-taggar med name-attribut samt http-equiv-taggar.

Meta-taggar med name-attribut:

description

En sträng som används för att beskriva ens hemsida. Denna information används av sökmotorer när man söker efter information. Beskriver man vad som finns på sin hemsida kan man möjligtvis få fler besökare. 

            
        <meta name="description" content="Webbutveckling Klientsidan">

keywords

Här skriver man kommaseparerade nyckelord som representerar hemsidans innehåll. 

            
        <meta name="keywords" content="meta,taggar,HTML">

author

Här skriver man namnet på författaren som skapat hemsidan. 

            
        <meta name="author" content="Oskar Emilsson">

generator

En sträng-representation som representerar information den eventuella programvara som man använt för att generera ett HTML-dokument. Denna meta-taggen används inte om man exempelvis skapat sitt dokument i en texteditor. 

            
        <meta name="generator" content="Frontweaver 8.2">

Exempel-generator-tag hämtad från W3 Document metadata

robots

Denna tag tillåter programmeraren att modifiera lite hur ens hemsida blir indexerad i (de flesta) sökmotorer. Exempelvis kan man definiera följande: 

            
        <meta name="robots" content="noindex">

Detta säger åt en sökmotor att man inte vill att ens hemsida ska visas i sökresultaten. “robots" syftar till alla sökmotorer. Vill man konkretisera sig på att man inte vill att ens sida ska visas på just Google, men i andra sökmotorer, kan man istället skriva: 

            
        <meta name="googlebot" content="noindex">

http-equiv-taggar

Dessa taggar skiljer sig en aning från meta-taggarna med name-attribut. Man kan använda sig av http-equiv om man vill skicka information till servern i form av en HTTP-header. Exempelvis kan man defininera att ens sida ska refreshas i ett visst tidsintervall via: 

            
        <meta http-equiv="refresh" content="30"> //Motsvarar omladdning var 30e sekund

Kort sagt så är name-taggar egentligen endast information, medan http-equiv-taggarna även kan innehålla instruktioner till webbläsaren som ovan visat. Dessa taggar bör användas med försiktighet då man, via bland annat automatisk omladdning av hemsidan, tar ifrån klienten kontrollen av hur man använder hemsidan.
