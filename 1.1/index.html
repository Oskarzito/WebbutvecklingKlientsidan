Doctype

<!DOCTYPE html> definierar doctype för dokumentet. Detta berättar för browsern (webbläsaren) vilken version av HTML dokumentet är skrivet i. Beroende på doctype:en så kan webbläsaren rita/rendera hemsidan på olika sätt.

Förut hade HTML väldigt långa (och svåra att komma ihåg) doctype-deklarationer. Exempelvis var man tvungen att skriva:

<!DOCTYPE html PUBLIC “-//W3C//DTD XHTML 1.0 Transitional//EN” “http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional-dtd”>

Teckenkod

Teckenkodningen för ett dokument talar om vilka tecken dokumentet är skrivet i. Exempelvis så används vanligtvis en teckenkodning som kallas för UTF-8, en teckenkodning som omfattar och representerar en uppsättning av alla Unicode-tecken.

Genom att deklarera teckenkodningen för ett dokument kan webbläsare tolka dokumentet på rätt sätt. Detta är väldigt bra om man förväntar sig internationell trafik på ens hemsida. Vidare, om det råkar vara så att man är ute efter att använda en uppsättning tecken som inte är så vanliga (till skillnad från UTF-8) är det extra viktigt att man definierar detta, annars måste webbläsaren reda ut det själv, vilket kan sakta ned tiden det tar för en sida att ladda.

Språk *(Antar att detta syftar främst till programmeringsspråk)*

Ett språk kan simplast beskrivas som en uppsättning maskininstruktioner som producerar någon form av resultat/utdata (output).

Det finns olika språk som vardera kan vara bra på olika saker. Låt oss avgränsa detta lite till kursens tre främsta språk: HTML, CSS samt JavaScript. Dessa tre samarbetar ofta för att ge en klient den bästa upplevelsen av en hemsida. En liten analogi skulle vara om vi föreställer oss en varelse. Då hade HTML varit varelsens skelett (underliggande strukturen), CSS hade varit varelsens hud/päls (dess yttre utseende), och JavaScript hade utgjort varelsens personlighet (hur den beter sig).

I HTML kan man även indikera ens hemsidas "naturliga språk". Genom att definiera detta vet webbläsaren att språket på hemsidan är exempelvis svenska som denna sida.

Validering

Validering handlar lite om att “korrekturläsa” något. Vi kan exempelvis validera kod, såsom HTML- och CSS-kod. I valideringsprocessen kontrolleras koden för bland annat syntax- samt strukturfel. En bra validator ger tips/meddelanden om vad som är fel och varför (och kanske till och med tips på hur man istället bör göra). Genom validering kan man kvalitetssäkra sina hemsidor.

För att validera HTML-kod kan man använda sig utav: W3 HTML Validator.
Och för att validera CSS-kod kan man testa: W3 CSS Validator.

Hur ett HTML-dokument är uppbyggt

Ett HTML-dokument är uppbyggt i en trädliknande struktur. Till att börja med deklarerar man doctype för dokumentet som kan säga till webbläsaren att “jag är ett HTML5-dokument”. Direkt efteråt så följer en öppnande och en stängande HTML-tagg som ser ut som följande:

<html> [massvis av HTML-kod] </html>

Det är mellan dessa två taggar man skriver resten av sitt dokument. Detta utgör därav roten till trädet, vilken även kommer kallas för ancestor till resterande taggar.

Vidare så följer två HTML-taggar som kallas för head (huvud) och body (kropp). Huvudet utgörs också av öppnande och stängande taggar. Likaså kroppen. Eftersom dessa båda ligger efter varandra (i koden), och båda är barn till samma ancestor, kallas dessa för syskon.

Huvudet i ett HTML-dokument innehåller information gällande själva dokumentet. Däribland exempelvis teckenkodningen, hur webbläsaren ska hantera dokumentet, vilka CSS- samt JavaScript-filer som ska länkas in (via link-taggar) och så vidare. Som exempel kan man i nedanstående HTML-dokument se att titeln definieras i dokumenthuvudet.

Kroppen i sin tur består däremot av det innehåll som klienterna ser i sin webbläsare. Detta kan vara bland annat knappar, bilder, länkar text och så vidare. Det lilla HTML-dokumentet nedan visar exempelvis en länkad rubrik följt av lite text. Det är sedan taggarna i kroppen som man modifierar och style:ar i sina CSS-filer samt ger ett beteende i sina JavaScript-filer. 

                
        <!DOCTYPE html>
        <html>
            <head>
                <title>Grunderna i HTML5</title>
            </head>
            <body>
                <h1>
                    <a href=”#”> Rubrik </a>
                </h1>
                <p>Lite text</p>
            </body>
        </html>
