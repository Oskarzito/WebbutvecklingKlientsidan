1.3 Semantiska element för text

I HTML finns det intressanta semantiska element för text. Här går vi igenom time, datetime, mark samt wbr.

time och datetime

I HTML kan det vara av intresse att visa tider. Detta kan man göra med ett HTML-element som heter time. time är tänkt att vara till för att människor ska läsa det. Exempelvis kan vi skriva: "Vi ses på julafton 18:00". Detta blir en prydligt utskriven tid på skärmen. 

            
                <p>Vi ses på julafton <time>18:00</time></p>

Till skillnad från endast time finns ett attribut för HTML-element kallat datetime. datetime är inte till för människor, det är till för att maskiner ska läsa tider. Skriver man i sin källkod: 

            
                <p>Vi ses på <time datetime="2018-12-24 18:00">julafton</time></p>

blir sedan resultatet på skärmen:

Vi ses på julafton

Här ser vi att tiden 18:00 inte visas. Detta syns dock om en maskin skulle läsa det. Eftersom vissa specifika format för tider och datum följs kan detta lätt hanteras.

wbr

wbr står för word break opportunity. Detta indikerar i koden var det är lämpligt att radbryta ett ord om ordet skulle vara för långt för att visas på skärmen. Lånar ett exempelord från W3 schools för att visa hur funkar om man minskar/ökar storleken på webbläsarfönstret.

This is a veryveryveryveryveryveryveryveryveryveryveryveryveryveryveryveryveryverylongwordthatwillbreakatspecificplaceswhenthebrowserwindowisresized.

Detta ord har mitt i sig, där man föreslår att ordet radbryts, den inline:ade taggen <wbr>.
"...veryvery<wbr>longword...specific<wbr>places...". Där taggen definieras kommer ordet radbrytas vid behov.

mark

Taggen mark markerar/highlightar den text man skriver inom elementets öppnande och stängade taggar, vilket kan vara behändigt om man vill betona något.
