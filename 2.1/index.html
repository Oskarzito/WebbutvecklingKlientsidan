Externt länkad CSS-fil och dess fördelar

CSS kan appliceras på HTML-kod på främst två sätt, inline och externt. Fördelarna med att använda externt länkade CSS-filer är flertalet. Här är några:

  • Man separerar helt HTML-kod från CSS-kod. Kodseparation gör det enklare att underhålla sin applikation. Flera HTML-filer kan exempelvis använda sig av en enda CSS-fil för stilsättning.
  • Ens CSS-kod finns på ett ställe. Detta gör att när man uppdaterar stil-attribut kommer allt som använder sig av den CSS-koden uppdatera sitt utseende.
  • Hemsidor laddar snabbare om de exempelvis cachar en CSS-fil.
  • Extra: rent generellt skulle jag personligen säga att det är enklare att samla ens CSS-kod i en fil då detta lättare gör att man kan fokusera på det man gör. Till exempel, när man skrivet sin HTML fokuserar man endast på de man vill ha på skärmen. När man sedan skriver sin CSS fokuserar man endast på utseendet och så vidare.

Arv

Arv i CSS har sitt ursprung i HTML:s struktur. Då HTML har en trädliknande struktur kommer stilsättningen av dess element påverkas. Stilsätter vi ett HTML-element kommer detta elements barn att få samma stil-attribut – de ärver stilen. Barn ärver stilen från sina föräldrar.

Cascade

Cascade handlar, till skillnad från arv, om vad för stil ett HTML-element ska anta. Om vi skriver: 

                        
                        p {
                            font-size: 50px;
                        }

kommer alla paragrafer få font-storleken 50 pixlar. Om vi sedan skriver: 

                        
                        p {
                            font-size: 50px;
                        }

                        p.enKlass {
        	                font-size: 100px;
                        }

kommer alla paragrafer få font-storleken 50 pixlar, men alla paragrafer som tillhör klassen “enKlass” får storleken 100 pixlar. Cascade handlar alltså om vilka stilsättningsregler som får precedens över andra regler vid eventuella stilsättningskonflikter.

Selektorer (tagg-, attribut-, klass-, id- och barn-selektorer)

Selektorer är det man använder för att välja ut vad man ska stilsätta.

  • Tagg syftar till en HMTL-tagg. Här kan vi exempelvis stilsätta alla h2-rubriker genom att säga: 

                                    
                    h2 {
                        /*CSS-kod*/
                    }

  • Attribut-selektorn väljer ut särskilda HTML-element som har en viss egenskap (ett attribut). Exempelvis kan vi stilsätta a-länkar enligt följande: 

                                
                    a { /*Stilsättning via tagg-selektorn*/
                	    background-color: red;
                    }

                    a[target=”_blank”] {
                        /*Stilsättning via attribut-selektorn*/
                	    background-color: blue;
                    }

    Här väljer vi ut (selekterar) alla a-taggar i och ger dessa en röd bakgrund. Det vi sedan gör är att vi väljer ut alla a-taggar som innehar attributet target=”_blank”, vilket betyder att man specificerat att de vid klick ska öppnas i en ny webbläsarflik.

  • HTML-taggar kan tillhöra en viss klass via att man definierar följande: <p class=”myClass”> eller <h1 class=”myClass”> och så vidare. Självklart kan man välja att stilsätta element som tillhör samma klass via: 

                                    
                    .myClass {
                    	text-align: center;
                    }

    vilket gör att att alla HTML-element som tillhör klassen myClass kommer centreras i mitten. Klasser väljs ut genom att skriva en punkt följt av klassnamnet.

  • Likt klass-selektorn kan man även välja ut element med ett visst id. Om vi har <p id=”myPara”> kan vi välja ut detta element via: 

                                    
                    #myPara {
                    	font-size: 15px;
                    }

    Genom att skriva ett #-tecken följt av id-namnet selekterar vi efter id.

  • Barn-selektorer väljer ut HTML-element som är barn till andra HTML-element. 

                                    
                    div > p {
                    	font-size: 15px;
                    }

    Detta väljer ut alla p-element vars förälder är ett div-element.

Enheter (absoluta: in, cm, mm, pt, pc, px och relativa: em, ex, %)

Absoluta enheter är syftar till exakta mått/storlekar. Man anger här explicit vilken storlek man vill att något ska vara. Samma angivelse av absoluta enheter på två element kommer göra dessa lika stora.

  • in: inches (2.54 cm).
  • cm: centimeter.
  • mm: millimeter.
  • pt: points (1/72-dels inch).
  • pc: picas (12 points).
  • px: pixels.

Relativa enheter är beroende av storleken på något annat. Det vill säga, att ange två likadana relativa storlekar på två olika element gör inte nödvändigtvis dessa till lika stora.

  • em: relativ till nuvarande font. Antal em är antal gånger större än nuvarande font.
  • ex: relativ till x-höjden av exempelvis nuvarande font.
  • %: en viss procent av förälder-elementets storlek.

Hur ett CSS-dokument är uppbyggt

Ett CSS-dokument består av stilsättningsregler. Reglerna ser ut som enligt följande: 

                        
                    selektor {
                        attribut: värde;
                    }

Selektorn ovan omfattar allt som har med att välja ut vad man vill stilsätta. Det kan vara taggar, klasser, id eller relationer mellan element. När man valt ut något skrivs sedan två måsparanteser. Detta är stilsättningens scope för vald selektor. Det är mellan dessa paranteser man definierar vad som ska stilsättas och hur. Genom att skriva attributet, följt av ett kolon, följt av ett värde, följt av ett semikolon radar man upp stil efter stil. Det får mer än gärna finnas flera stilsättningar i samma scope. Man anger bara igen, på ny rad, vilket attribut som ska stilsättas och sedan hur. Ett attribut kan även ta flera värden. Exempelvis tar attributet flex: tre värden – öka, minska samt hur man ska beräkna ny storlek.

Känner man för att välja ut något annat att stilsätta kan man fortsätta på ny rad efter den stängade måsparantesen på samma sätt som ovan. Man radar helt enkelt upp stilregel efter stilregel för att formge sin hemsida.
